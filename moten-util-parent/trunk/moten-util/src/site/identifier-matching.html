Identifier Matching Function

Draft

Mathematical description of a an identifier matching function based on identifiers with types that are comparable.

This document is best viewed with Firefox or another SGML compliant browser. IE has problems with the mathematical characters up to version 7.

Abstract

A set of entities with non-conflicting identities is represented by a set of disjoint identifier sets. Each identifier set has an associated time. The problem is to define a function that takes a new set of identifiers representing an update on one entity with it's associated time and merges it with the existing entity set.

If the new identifiers do not exist in the entity set then a new entity is created with those identifiers. If one or more of the new identifiers do exist in the entity set (possibly against multiple entities) then merge decisions need to be made.

Find a primary match to the set. Apply all identifiers to that primary match, replacing identifiers of the same type if the new identifiers time is greater than the primary match time.

Now take the resultant changed primary match and apply a merge function with every identifier set that has a common identifier with it.

The merge function will accept two identifier sets. If the first set is the empty set or the second set is stronger than the first set return the empty set. Otherwise, for every identifier in the second identifier set add the identifier to the first set if the identifier type does not exist in the first. If the identifier type does exist in the first then replace the identifier in the first set with the identifier type with the second identifier iff the second timestamp is after the first time. Reversing the parameters will return the adjusted secondary matching identifier set.

Identifier Matching Function

Define

I = set of identifiers

T = set of identifier types

S = set of timed identifiers = I ⨯ ℜ

For x ∈ S, x = (xid, xtime) where xid ∈ I, xtime ∈ ℜ

B = {true, false}

Given

t: I → T, t: S → T categorise an identifier with an identifier type

> : TxT → B is an infix comparator of the strength of identifier types

>> : TxT → B is an infix comparator of the order of identifier types where

∀ s,t ∈ T, (s !> t and t !> s) ⇔ s = t, s > t ⇒ s >> t

Define

c (conflicting identifiers) : ℘(S) → ℘(S) given by

c(X,Y) = {x ∈ X : t(xid) ∈ t(Yid) and xid ∉ Yid}

V (valid identifier sets) = { X ⊂ S : c(X,X) = ∅}

Z' = { Z ⊂ ℘(V) where elements of Z are disjoint sets}

pm (primary match) : Z' ⨯ S → S given by

pm(Z,X) =

∅     , Z = ∅ or Xid ∩ Yid = ∅ ∀ Y ∈ Z

Y ∈ Z where Yid ∩ Xid >> Rid ∩ Xid ∀ R ∈ Z, R ≠ Y     , otherwise

γ : S ⨯ S → S by

γ(X,Y) =

{x ∈ X : t(x) ∉ t(Y) }

G : S ⨯ S ⨯ → S by

G(X,Y) =

γ(Y,X) ∪ { i ∈ Y : ∃ j ∈ X s.t. t(i) ∈ t(X) and itime > jtime }

G(X,Y) is thus the subset of Y that has no type in X or those that match type with X but have a more recent time.

Gns : S ⨯ S ⨯ B → S by

Gns(X,Y) =

γ(Y,X) ∪ { i ∈ Y : ∃ j ∈ X s.t. t(i) ∈ t(X) and itime ≥ jtime }

G(X,Y) is thus the subset of Y that has no type in X or those that match type with X but have a more recent time.

⊗: S ⨯ S ⨯ S → S by

X ⊗R Y =

{ x ∈ X : xid ∉ Yid}     , tmax(Y) > tmax(R)

γ(X,G(X,Y)) ∪ G(X,Y)     , otherwise

β : S → S by

β(X,Y) = X ∪ {y ∈ Y : t(y) ∉ X }

f: Z' ⨯ S → Z' by

f(Z,A) =

fold( {Y ∈Z : Yid ∩ Aid ≠ ∅},⊗A , β(pm(Z,A),A) ⊗A A)

Finally, define α: Z' ⨯ S → Z' by

α(Z,A) =

Z ∪ {A}     , pm(Z,A) = ∅

{fns(A,X) : X ∈ Z, Xid ∩ Aid ≠ ∅, Gns(A,X) ⊄ f(Z,A)} ∪ {X ∈ Z : Xid ∩ Aid = ∅} ∪ {f(Z,A)}     , otherwise

α(Z,A) is thus the entity set obtained by merging timed identifier set A into entity set Z.

Notes

>> is an arbitrary comparator whose only purpose is to ensure deterministic behaviour of α.
